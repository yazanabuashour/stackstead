# Stackstead 0.1.4

Stackstead 0.1.4 is a reliability release for parallel lifecycle entry,
diagnosis, interrupted agents, and retryable exact teardown. It also adds direct
command execution inside an owned Compose service.

## Highlights

### Lifecycle operations wait instead of failing immediately

Concurrent lifecycle commands now use a bounded wait on the existing project
and environment locks. A queued mutation revalidates teardown and identity state
after acquiring the lock instead of relying on a stale pre-wait observation.

### Teardown is resumable

Destroy records one identity- and token-bound phase journal. A retry resumes
`runtime_remove`, `source_remove`, or `finalize` without rerunning a completed
hook or releasing the global port lease early. When ordinary source removal
cannot delete container-created files, a narrowly attested helper removes only
the exact Stackstead-owned worktree.

### Interrupted agents release their environment safely

On Unix, `stackstead run` supervises the target process group and retains the run
lease until cleanup completes. Linux additionally reaps detached descendants.
macOS supports exact process-group cleanup; descendants that deliberately create
a new session remain best effort.

### Inspection reports effective live status

`StacksteadInspection` version 3 separates durable recorded state, observed live
state, and the effective status automation should use. Recorded/live divergence
produces an explicit warning, and a failed teardown has its own effective phase.

### Run commands inside a Compose service

```sh
stackstead exec <full-id> <service> -- <command> [args...]
```

`exec` verifies the exact owned running service, uses the generated Compose
contract, holds the run lease for the Compose client lifetime, and preserves the
child exit status.

## Additional changes

- `doctor` reports missing or stale Stackstead repository policy markers.
- Compose and lifecycle validation fail closed on more redirected, ambiguous,
  malformed, or tampered state.
- Installation and agent setup documentation are shorter and point to the
  checksummed immutable release flow.
- CI enforces formatting, warnings, safe Rust lint policy, packaging, delivery,
  release-install, macOS, and live Docker integration checks.

## Upgrade notes

The 0.1.3 inspection, hook, and teardown-recovery contracts do not have a
mixed-version bridge. Follow the complete
[0.1.3 to 0.1.4 upgrade procedure](../upgrade-0.1.4.md) before replacing the old
binary or trusted manager hooks. It includes exact-target recovery for an
already failed 0.1.3 teardown.

Stackstead is pre-1.0 and does not provide compatibility bridges for earlier
experimental teardown journals or older inspection transports. Command-owned
JSON consumers must validate `kind` and `version` before reading fields.

## Install

```sh
curl -fsSL https://github.com/yazanabuashour/stackstead/releases/download/v0.1.4/install.sh \
  | sh -s -- --version 0.1.4
stackstead --version
```

The release provides checksummed binaries for Intel and Apple Silicon macOS and
x86-64 and ARM64 glibc Linux. Windows and musl binaries are not published.

See [Reliability evidence](../reliability.md) for the dogfood failures, focused
rerun, and remaining limits.
